Overview


NAS (Network Algorithms Simulator) enables you to simulate various algorithms. You can create new networks, open existing networks, execute the algorithms, simulate an adversary by modifying the network and the nodes it composed from, etc.

As seen in the image above, the user interface is composed of the following sections:

  1. Menu Bar - Contains the various menus, enables most of the supported actions
  2. Tool Bar - Contains the various tool bars, enables most of the supported actions
  3. Network Panel - The workspace for composing the networks
  4. Views - Various additional views, arranged in tabs
All the sections above are described in details in the next topics. top

Tool Bars

The tool bar contains several separated tool bars, described below.

File

The File tool bar enables the following file actions:

  1. Create a new file (a new network)
  2. Open an existing file
  3. Save the current network

Edit

The Edit tool bar enables the following actions:

  1. Switch between view mode and edit mode
  2. Add a new node to the network panel
  3. Delete the selected element (node or connector)
  4. Switch between editing modes
    1. Move nodes - Drag & drop a node in order to change its location
    2. Connect nodes - In order to connect two nodes - click on a node, drag while still pressed, release over the node to connect to

Simulation

The Simulation tool bar enables the following actions:

  1. Undo the last action (only actions done by the executed algorithm)
  2. Perform the next step of the algorithm
  3. This is an indicator for the current "time" in the algorithm's execution

New Network Model Wizard

When you decide to create a new network model, you will get to the dialog seen above. Every network model is associated with an algorithm. This is because each algorithm requires different node variables. This dialog lets you select the algorithm for which you would like to create the network.

As you can see, the dialog displays a list of all available algorithms. Select an algorithm in order to see a more detailed description of it. In order to create the network, approve the selection.

top

Algorithm Execution

After a network was composed, you can simulate the algorithm's execution on it. In order to do so, switch to "View Mode" and use the simulation actions. During the execution, each node can be annotated according to the implementation of the specific algorithm. Additional information can be viewed in the Console view. You can look at the simulation tool bar for indication on the current "time" of execution.

In order to simulate an adversary, switch back to "Edit Mode", select the "Variables" view, and modify the variables (according to the definition of the specific algorithm).
For more information, see the information on the specific algorithms.


  i   Note
After any manual modification of the network (add/remove/connect/disconnect nodes), or change to the values of a node's variables, the steps cannot be undo.

Views (Tabs)

Following are several views for additional information or actions.

Console

The console displays messages from the application. Most of the messages are related to the simulation process. During the execution of an algorithm, you can watch the console in order to see what exactly is done in each step.

Variables

The variables view displays the various variables of the selected element in the network. The displayed variables are pairs of variables name and its value. The view has two modes: View & Edit. In order to switch between the two modes, use the button in the Edit tool bar. When in edit mode, the values of the variables can be modified (whether for setup proposes, or simulating an adversary).

Algorithm

The Algorithm view displays the information on the algorithm that the current network model is associated to. This information is similar to the information given in the "New Network Model Wizard". It should contain the algorithm's name, basic description of it, and sometimes further hints on it.
